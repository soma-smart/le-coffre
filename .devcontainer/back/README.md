# Backend DevContainer Setup

This devcontainer provides a complete Python development environment with a persistent Keycloak instance for integration testing.

## Features

- **Python 3.13** with uv package manager
- **Persistent Keycloak** instance (runs as a separate service)
- **Pre-configured test environment** with test user and OAuth client
- **VS Code extensions** for Python development (Pylance, Ruff, Debugpy)
- **Dynamic user mapping** (files created in container have correct host permissions)

## Keycloak Configuration

The devcontainer automatically sets up Keycloak with:

- **Keycloak URL**: `http://keycloak:8080`
- **Admin Console**: `http://localhost:8080` (accessible from host)
  - Username: `admin`
  - Password: `admin`
- **Realm**: `master`
- **Test Client ID**: `lecoffre-test-client`
- **Test User**:
  - Username: `testuser`
  - Password: `testpass123`
  - Email: `testuser@example.com`


## Keycloak Configuration Details

The configuration is stored in `/tmp/keycloak-test-config.env` and includes:

```env
KEYCLOAK_URL=http://keycloak:8080
KEYCLOAK_REALM=master
KEYCLOAK_CLIENT_ID=lecoffre-test-client
KEYCLOAK_CLIENT_SECRET=<generated-secret>
KEYCLOAK_TEST_USERNAME=testuser
KEYCLOAK_TEST_PASSWORD=testpass123
KEYCLOAK_TEST_EMAIL=testuser@example.com
```

## Reconfiguring Keycloak

If you need to reset or reconfigure Keycloak, run:

```bash
bash /app/.devcontainer/back/setup-keycloak.sh
```

This will:
- Wait for Keycloak to be ready
- Create/reset the test user
- Create/update the OAuth client
- Save the configuration to `/tmp/keycloak-test-config.env`

## Troubleshooting

### Keycloak not ready

If tests fail because Keycloak is not ready:

```bash
# Check Keycloak health
curl http://keycloak:8080/health/ready

# Re-run the setup script
bash /app/.devcontainer/back/setup-keycloak.sh
```

### Missing client secret

If tests are skipped due to missing client secret:

```bash
# Run the setup script to generate configuration
bash /app/.devcontainer/back/setup-keycloak.sh

# Verify configuration exists
cat /tmp/keycloak-test-config.env
```

### Access Keycloak admin console

1. Open `http://localhost:8080` in your browser (from host machine)
2. Login with `admin` / `admin`
3. Navigate to the `master` realm
4. View users and clients

## Environment Variables

The following environment variables are available:

- `KEYCLOAK_URL` - URL of the Keycloak instance (default: `http://keycloak:8080`)
- `KEYCLOAK_REALM` - Realm name (default: `master`)
- `KEYCLOAK_CLIENT_ID` - OAuth client ID (default: `lecoffre-test-client`)
- `KEYCLOAK_CLIENT_SECRET` - OAuth client secret (generated by setup script)
- `KEYCLOAK_TEST_USERNAME` - Test user username (default: `testuser`)
- `KEYCLOAK_TEST_PASSWORD` - Test user password (default: `testpass123`)
- `KEYCLOAK_TEST_EMAIL` - Test user email (default: `testuser@example.com`)
